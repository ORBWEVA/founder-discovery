CUSTOMER DISCOVERY COMMANDS
=============================
Talk to users before you build.

Command                  | What it does
─────────────────────────┼─────────────────────────────────────────────
/discovery:interview     | Mom Test interview scripts + question banks
/discovery:survey        | Survey design with bias prevention
/discovery:jobs          | Jobs-to-be-Done mapping + switch analysis
/discovery:analyze       | Synthesize research into actionable insights
/discovery:help          | This help screen

RECOMMENDED FLOW
  1. /discovery:interview  → Talk to 10-20 users
  2. /discovery:survey     → Quantify what you heard
  3. /discovery:jobs       → Map the job they're hiring you for
  4. /discovery:analyze    → Turn data into decisions

PAIRS WITH
  /dt:*           → Design Thinking (empathize, define, ideate)
  /gtm:*          → Go-To-Market (positioning, channels, pricing)
  /startup:*      → Startup Metrics (PMF, unit economics, benchmarks)

Install: npx skills add ORBWEVA/founder-discovery
Docs:    https://github.com/ORBWEVA/founder-discovery
